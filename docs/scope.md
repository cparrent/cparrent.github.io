# Project Scope Statement

## Project Title

Law Enforcement Professional Portfolio Website

## Project Purpose

The purpose of this project is to develop a professional portfolio website that highlights my experience, qualifications, certifications, and career goals in law enforcement. The website will serve as an online portfolio that demonstrates both my professional background and the web development and project management skills learned throughout this course.

## Project Objectives

The project will:

- Create a professional website using HTML and CSS.
- Present my law enforcement background, certifications, and professional experience in a clear and organized manner.
- Demonstrate effective project planning using the principles from *Project Management for the Unofficial Project Manager (PMUPM)*.
- Publish the completed website using GitHub Pages.
- Produce a well-documented GitHub repository that showcases both technical and project management skills.

## Project Deliverables

The completed project will include:

- A public GitHub repository.
- A professional `README.md` explaining the project and linking to the live website and project documentation.
- A `docs/` folder containing:
  - `scope.md`
  - `plan.md`
  - `retrospective.md`
- `index.html`
- `about.html`
- `style.css`
- A live website hosted through GitHub Pages.

## Project Scope

### In Scope

The project includes:

- A professional homepage introducing me and my law enforcement career.
- An About page describing my background, experience, and career goals.
- A consistent navigation menu linking both pages.
- A clean, professional layout using a single external CSS stylesheet.
- Sections highlighting:
  - Professional summary
  - Law enforcement experience
  - Specialized training and certifications
  - Skills and qualifications
  - Career objectives
- Appropriate color palette that support the professional theme.
- Complete project documentation required by the course.

### Out of Scope

The project will **not** include:

- User accounts or login functionality.
- Online employment applications.
- Interactive databases.
- JavaScript-based applications.
- Discussion forums or comment sections.
- Real-time communication features.
- Server-side programming.
- Storage or display of sensitive law enforcement information or confidential records.

## Success Criteria

The project will be considered successful if:

- The website is publicly available through GitHub Pages.
- Navigation between pages works correctly.
- The design is professional, organized, and easy to navigate.
- The HTML and CSS are well-structured and consistently applied.
- The project documentation is complete and follows assignment requirements.
- The GitHub repository demonstrates good organization and version control.
- All Project 03 requirements have been satisfied.

## Constraints

### Time

The project must be completed before the course deadline. Time will be allocated for planning, website development, testing, documentation, revisions, and final deployment.

### Budget

The project will be completed using free development tools and hosting services, including GitHub, GitHub Pages, Visual Studio Code, HTML, and CSS. No additional financial resources are expected.

### Resources

The project will be completed by a single developer using course materials, PMUPM, GitHub documentation, and publicly available web development references.

### Technical Constraints

The website must use HTML and CSS as required by the course. All pages will share one external stylesheet and remain compatible with current web browsers. The website must also comply with the project requirements established for Project 03.

## Assumptions

The project assumes:

- GitHub Pages will remain available for hosting.
- Internet access will be available throughout development and submission.
- Modern web browsers will properly display the website.
- Course requirements will remain unchanged during the project.
- Professional information included in the portfolio is accurate and appropriate for public viewing.

## Stakeholders

- **Project Manager:** Student Clayton Parrent (Project Owner)
- **Primary Stakeholder:** Course Instructor Cheri Whalen, M.S.
- **End Users:** Potential employers, law enforcement agencies, classmates, and other visitors interested in viewing the professional portfolio.

## Risks

Potential risks include:

- Delays caused by limited development or testing time.
- HTML or CSS errors affecting the website's appearance.
- GitHub Pages deployment issues.
- Scope changes requiring additional revisions.
- The need to balance professionalism with the amount of information presented so the portfolio remains concise and easy to navigate.

## Acceptance Criteria

The project will be accepted when:

- The GitHub repository is public and professionally organized.
- The README includes a project overview and links to both the live website and project documentation.
- The website contains the required HTML pages and a shared external stylesheet.
- Navigation functions correctly across the website.
- The design presents a professional law enforcement portfolio.
- All required documentation is included in the `docs/` folder.
- The website is successfully deployed through GitHub Pages.
- All Project 03 assignment requirements have been met.
