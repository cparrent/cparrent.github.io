# Project Plan

## Project Overview

This project plan outlines the approach for developing the **Law Enforcement Professional Portfolio Website**. The project will be completed by following the project management principles presented in *Project Management for the Unofficial Project Manager (PMUPM)*. The plan identifies potential risks, strategies for managing those risks using the TAME framework, and the major tasks required to successfully complete the project.

---

# Risk Analysis (TAME Framework)

| Risk | Probability | Impact | TAME Strategy | Mitigation Plan |
|------|-------------|--------|---------------|-----------------|
| Running out of time before the project deadline | Medium | High | Mitigate | Create a development schedule, complete tasks in small milestones, and leave time for testing and revisions. |
| HTML or CSS coding errors | Medium | Medium | Mitigate | Test the website after completing each section, validate HTML and CSS, and fix issues before moving to the next task. |
| GitHub Pages deployment problems | Low | High | Transfer | Use GitHub documentation, course resources, and instructor guidance if deployment issues occur. |
| Scope changes requested after development begins | Low | Medium | Accept | Evaluate the requested changes, determine their effect on time and quality, and implement only if they fit within the project schedule. |
| Browser compatibility or formatting issues | Medium | Medium | Mitigate | Test the website using multiple web browsers and different screen sizes before submission. |

# Work Breakdown Structure (WBS)

## Phase 1 – Project Planning

- Review Project 03 requirements.
- Finalize the Project Scope Statement.
- Create the project repository on GitHub.
- Organize the project folder structure.

## Phase 2 – Website Development

- Create `index.html`.
- Create `about.html`.
- Build the navigation menu.
- Add professional portfolio content.
- Apply a consistent color palette and layout using `style.css`.

## Phase 3 – Testing and Quality Assurance

- Test navigation links.
- Validate HTML and CSS.
- Review formatting and readability.
- Test the website in multiple browsers.
- Verify the website displays correctly on different screen sizes.

## Phase 4 – Documentation

- Create `README.md`.
- Complete `plan.md`.
- Complete `retrospective.md`.
- Review all documentation for accuracy and completeness.

## Phase 5 – Deployment

- Upload all project files to GitHub.
- Publish the website using GitHub Pages.
- Verify the live website functions correctly.
- Perform a final review to ensure all project requirements have been met.

| Milestone | Expected Outcome |
|-----------|------------------|
| Project Planning Complete | Scope statement approved and repository created. |

| Testing Complete | Website functions correctly and passes validation checks. |

# Success Plan

The project will be considered successful if:

- The website is completed by the course deadline.
- All required project files are included in the GitHub repository.
- Navigation works correctly between all pages.
- The design is professional, organized, and easy to use.
- Documentation accurately reflects the project planning process.
- The website is successfully deployed through GitHub Pages.
- All Project 03 assignment requirements have been satisfied.
