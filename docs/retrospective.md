# Project Retrospective

## Project Overview

The Law Enforcement Professional Portfolio Website was developed to demonstrate my web development skills while creating a professional online portfolio highlighting my experience, qualifications, certifications, and career goals in law enforcement. Throughout this project, I applied the project management principles from *Project Management for the Unofficial Project Manager (PMUPM)* to plan, organize, develop, and deliver the project.

---

# What Went Well?

One of the biggest successes of this project was starting with a detailed Project Scope Statement. Defining the project's purpose, objectives, deliverables, and constraints early helped keep the project organized and prevented unnecessary scope changes during development.

Another success was maintaining an organized GitHub repository. Separating the documentation into the `docs` folder and keeping the website files in the root directory made the project easier to manage and review. Using GitHub also provided version control, allowing changes to be tracked throughout the project.

Developing the website using a single external CSS stylesheet ensured that both pages maintained a consistent appearance. The final design reflected a clean and professional layout that supports the purpose of a law enforcement portfolio.

---

# What Went Wrong?

The most challenging part of the project was balancing development time with testing and documentation. Creating the website itself required less time than ensuring the HTML, CSS, and documentation all met the project requirements.

Another challenge was organizing the project so that every document aligned with the original scope statement. Small changes to the website sometimes required updates to the documentation to maintain consistency throughout the project.

Testing the website across different browsers and screen sizes also required additional time to ensure that the layout remained professional and easy to navigate.

---

# What Would I Do Differently?

If I were completing this project again, I would begin testing earlier in the development process instead of waiting until most of the website was complete. Identifying formatting issues earlier would reduce the amount of time spent making corrections near the end of the project.

I would also commit changes to GitHub more frequently. Smaller, more frequent commits would create a clearer development history and make it easier to track progress throughout the project.

Finally, if additional time were available, I would expand the portfolio by adding more pages, such as a dedicated certifications page, training history, professional achievements, and a contact page. These additions would provide visitors with a more complete overview of my law enforcement background while maintaining the professional design established in this project.

---

# Lessons Learned

This project reinforced the importance of planning before beginning development. Creating a clear scope statement, identifying project risks, and following a structured project plan made the development process more organized and manageable. I also gained additional experience using GitHub for version control, developing websites with HTML and CSS, and applying project management techniques to complete a project from initial planning through final deployment.

Overall, this project strengthened both my technical skills and my understanding of how effective planning contributes to a successful project outcome.
